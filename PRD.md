# 情侣旅行回忆分享网站 PRD v1.0

## 1. 产品定位

本产品第一版定位为面向你们两人的私有情侣旅行故事站，通过 3D 数字地球、城市足迹、旅行路线、纪念日和沉浸式回忆内容，记录并分享两个人共同经历过的城市、旅行与重要时刻。

第一版核心服务对象是你们本人，以及通过分享链接访问的亲友。公共社区大厅仅作为后续方向入口，展示“即将开放/精选预览”，暂不开放完整多用户社区发布。

## 2. 产品目标

1. 用视觉冲击强的 3D 地球首页展示你们去过的城市。
2. 让访客可以从城市进入，查看该城市下的多段图文/视频回忆。
3. 支持按旅行组织内容，一次旅行可包含一个或多个城市。
4. 支持纪念日页面，记录恋爱、生日、第一次旅行等重要日期。
5. 支持选择性分享，亲友无需登录即可查看被开放的内容。
6. 为后续 PostgreSQL、对象存储、多情侣社区、精细权限和主题装扮扩展预留结构。

## 3. 用户角色

### 3.1 本人用户

你们两人，各自拥有账号，并通过情侣关系绑定。可登录后管理城市、旅行、回忆、纪念日、标签、草稿、发布状态、分享设置和访问统计。

### 3.2 亲友访客

通过分享链接访问，无需登录。只能查看已发布且被分享开放的城市、旅行、回忆和纪念日内容，不可搜索、编辑、评论或互动。

### 3.3 普通访客

访问网站公共入口时，可看到公共社区大厅的“即将开放/精选预览”。第一版不支持注册发布社区内容。

## 4. 信息架构

核心页面：

1. 首页：3D 地球仪、城市足迹点、城市标签、精选旅行故事/时间线入口。
2. 城市详情页：城市地图近景、该城市下的多段回忆卡片列表。
3. 回忆详情页：按时间顺序展示文字、图片、视频内容块。
4. 旅行详情页：地图路线连线、时间轴、经过城市、关联回忆。
5. 纪念日页面：倒计时卡片、重要纪念日列表。
6. 内容管理区：隐藏在头像/设置菜单中，仅本人登录可见。
7. 公共社区大厅：可点击入口，展示即将开放和精选预览。
8. 登录/注册页：手机号或邮箱注册登录，UI 预留第三方登录。

## 5. 核心功能需求

### 5.1 首页 3D 地球

首页展示 Cesium 风格的核心体验简化版 3D 数字地球。

功能要求：

1. 支持滑动旋转、缩放浏览。
2. 使用较真实的地球材质、光照和卫星底图风格。
3. 第一版只显示城市点位，不显示旅行路线连线。
4. 已去过城市显示为粉色光晕发光圆点和城市名称标签。
5. 点击城市圆点或标签后，先执行地球相机飞行动画，聚焦到城市近景，再进入城市详情页。
6. 提供低性能模式，可降低光效、地图细节或使用简化渲染兜底。

### 5.2 城市详情页

城市详情页用于承载某座城市下的所有回忆。

功能要求：

1. 页面顶部保留真实地图近景，支持拖拽和缩放探索。
2. 近景区域从首页地球飞行过渡后进入。
3. 下方展示该城市下的回忆卡片列表。
4. 一个城市可多次到访，因此可关联多篇回忆。
5. 回忆卡片默认按时间排序，同时支持手动置顶和自定义排序。
6. 城市支持草稿/发布状态、分享状态、软删除与恢复。

### 5.3 回忆详情页

回忆详情页为沉浸式内容页。

功能要求：

1. 内容以时间顺序叙事流呈现。
2. 支持文字、图片、视频穿插展示。
3. 第一版图片和视频直接使用原文件展示，不做压缩和缩略图生成。
4. 语音留言和 BGM 做字段及 UI 展示预留，第一版不做完整播放链路。
5. 每个内容块可单独记录天气、心情 Emoji 和标签。
6. 每篇回忆可选择独立视觉主题。
7. 第一版内置 2 个主题：浪漫梦幻、高级旅行影像杂志。
8. 回忆支持草稿/发布、分享开关、软删除与恢复。

### 5.4 旅行详情页

旅行用于组织一次完整旅程。

功能要求：

1. 一次旅行可包含一个或多个城市。
2. 旅行详情页展示地图上的城市路线连线。
3. 同时保留时间信息和时间轴。
4. 页面展示经过城市、旅行时间范围和所有关联回忆。
5. 旅行支持草稿/发布、分享开关、软删除与恢复。

### 5.5 纪念日页面

纪念日页面用于承载情侣关系中的重要日期，补充旅行之外的情感记录场景。

功能要求：

1. 页面入口放在主导航中，与首页、旅行、足迹、社区大厅并列。
2. 页面顶部展示重点纪念日倒计时卡片，例如恋爱纪念日、生日、下一次旅行、重要节日等。
3. 下方展示重要纪念日列表。
4. 每个纪念日包含标题、日期、类型、简短描述、封面图、倒计时或已过去天数。
5. 纪念日可关联城市、旅行或回忆，也可以单独存在。
6. 支持草稿/发布状态。
7. 支持分享开关。
8. 支持软删除与恢复。
9. 第一版不做复杂日历视图，后续可扩展月历、提醒、重复规则和通知。

### 5.6 内容管理

内容管理区只对本人登录后可见，入口隐藏在头像/设置菜单中。

功能要求：

1. 可新增、编辑、删除、恢复城市、旅行、回忆和纪念日。
2. 新增城市时，通过地图搜索选择城市，自动填充城市名与经纬度。
3. 新增回忆主流程：先选择或创建旅行，再选择或添加城市，最后编辑回忆内容。
4. 支持草稿/发布状态。
5. 支持回收站与恢复能力。
6. 第一版内容数据先 Mock，后续接 PostgreSQL 后允许真实编辑保存。

### 5.7 分享权限

第一版分享采用轻量模式。

功能要求：

1. 分享链接拿到即可访问，无需登录、密码或有效期。
2. 权限粒度第一版控制到城市和整篇回忆。
3. 纪念日也支持独立分享开关。
4. 后续预留内容块级权限能力。
5. 访客只能查看已发布且被分享开放的内容。

### 5.8 账号与情侣绑定

功能要求：

1. 两个人各自拥有账号。
2. 支持手机号或邮箱注册登录。
3. 第三方登录 UI 预留，第一版不接真实 OAuth。
4. 情侣绑定推荐使用邀请码方式。
5. 手机号/邮箱搜索并确认作为后续增强方向。

### 5.9 搜索与标签

搜索要求：

1. 本人登录后可使用搜索。
2. 支持全站统一搜索。
3. 城市列表、旅行列表、内容管理区支持局部搜索。
4. 第一版搜索范围包括城市名、旅行名、回忆标题和纪念日标题。

标签要求：

1. 支持预设标签库。
2. 允许新增自定义标签。
3. 标签可用于城市、旅行、回忆、纪念日或内容块。
4. 示例标签：美食、海边、纪念日、生日、第一次旅行。

### 5.10 访问统计

功能要求：

1. 第一版记录聚合数据。
2. 包括分享链接访问次数、城市浏览量、回忆浏览量、纪念日浏览量。
3. 不记录匿名访客明细日志、设备类型、来源页面等细粒度信息。

## 6. 非功能需求

### 6.1 性能

第一版优先保证视觉真实感，同时提供低性能模式。桌面端为第一优先级，移动端暂不作为第一版核心体验。

### 6.2 隐私

默认只有本人可管理内容。访客只能看到明确发布并分享的内容。

### 6.3 语言

第一版只支持中文界面。

### 6.4 部署

第一版按私有站点部署，不开放完整多情侣注册社区。

### 6.5 扩展性

数据模型需预留 PostgreSQL、对象存储、社区大厅、OAuth、细粒度权限、媒体处理、更多主题和多用户社区能力。

## 7. MVP 范围

第一版必须包含：

1. 3D 地球首页。
2. 城市点位与标签。
3. 点击城市后飞行聚焦并进入城市详情。
4. 城市详情页。
5. 回忆详情页。
6. 旅行详情页。
7. 纪念日页面。
8. 纪念日倒计时卡片和重要纪念日列表。
9. 登录注册基础 UI。
10. 情侣绑定基础设计。
11. 内容管理入口与 Mock 编辑能力。
12. 草稿/发布、分享、回收站。
13. 搜索、标签、访问统计。
14. 公共社区大厅占位页。

第一版暂不包含：

1. 完整多情侣社区发布。
2. 点赞、评论、留言祝福。
3. 真实语音播放链路。
4. BGM 完整播放链路。
5. 图片/视频压缩与缩略图生成。
6. 内容块级权限。
7. 第三方登录真实接入。
8. 移动端完整适配。
9. 纪念日月历、提醒、重复规则和通知。

## 8. 验收标准

1. 首页可流畅展示 3D 地球，并显示 10 多个城市点位。
2. 点击城市点位后，有相机飞行聚焦动画，并进入对应城市详情页。
3. 城市详情页可查看该城市下多篇回忆。
4. 回忆详情页可按时间顺序展示图文/视频内容块。
5. 旅行详情页可展示路线连线、时间轴和关联回忆。
6. 纪念日页面可正确展示倒计时和重要纪念日列表。
7. 本人登录后可进入内容管理区。
8. 本人登录后可新增、编辑、删除、恢复城市、旅行、回忆和纪念日。
9. 草稿内容不会被访客看到。
10. 分享链接可无需登录访问开放内容。
11. 搜索仅本人登录后可用。
12. 访问统计可展示聚合浏览数据。
